# Детальный алгоритм турнира Wine Competition

Этот документ описывает внутреннюю логику проведения голосования и турнира в приложении. Он предназначен для того, чтобы разработчик или организатор мог понять, как происходит жеребьевка, голосование и подсчет результатов, и при необходимости вручную поправить код.

Все основные механизмы турнира находятся в файле **`wine_app/views.py`**.

---

## 1. Запуск соревнования
**Функция:** `start_competition(request, competition_id)`
**Где смотреть:** `wine_app/views.py`

**Как работает:**
1. Проверяется, что соревнование находится в статусе `draft` (черновик) и в него добавлено **минимум 2 вина**.
2. Статус соревнования меняется на `running`.
3. В базе данных создается **Раунд 1** (`Round.objects.create(round_number=1, status='active')`).
4. Вызывается ключевая функция генерации турнирной сетки — `create_pairings(first_round)`.

---

## 2. Генерация пар (Жеребьевка)
**Функция:** `create_pairings(round_obj)`
**Где смотреть:** `wine_app/views.py`

Это самый важный алгоритм, отвечающий за расстановку вин в турнирную сетку.

### Раунд 1 (Старт турнира)
1. Выбираются все вина турнира и функция применяет случайное перемешивание `random.shuffle(wines)`. Случайная жеребьевка происходит **только один раз**, на старте.
2. Программа проверяет, равно ли количество участников степени двойки (2, 4, 8, 16). Если нет — применяется система **"Byes" (автоматических проходов)**:
   - Функция `get_next_power_of_2(n)` вычисляет ближайшую бОльшую степень (например, для 5 вин это будет 8).
   - `num_byes = 8 - 5 = 3`. Это значит, 3 вина пройдут во второй раунд без соперника ("пустышка").
   - `num_playing = 5 - 3 = 2`. Это значит, 2 оставшихся вина будут играть друг с другом в первой паре.
3. Составляются реальные пары (`wine1` vs `wine2`), и их статус ставится `pending`.
4. Составляются пары-пустышки (`wine2 = None`, победитель сразу назначается), и их статус сразу ставится `completed`.

### Раунды 2 и далее
1. Берутся победители предыдущего раунда: `wines = [p.winner for p in previous_round.pairings.all() if p.winner]`.
2. Функция просто берет их по порядку 1+2, 3+4 и создает новые пары (статус `pending`), **без случайного перемешивания**, поэтому победитель 1-й пары раунда-1 будет биться с победителем 2-й пары раунда-1.

---

## 3. Процесс проведения Пары
1. **Активация пары** (`start_pairing`): Организатор вручную нажимает кнопку "Запустить пару". Статус конкретной игры `Pairing` переходит в `active`. (Нельзя запустить пару, если уже есть другая активная).
2. **Окно голосования** (`competition_view`): Для пользователей открывается страница. Им назначается уникальная cookie `wine_competition_id`, по которой система будет отличать пользователей.

---

## 4. Как обрабатывается голос
**Функция:** `vote(request, pairing_id)`
**Где смотреть:** `wine_app/views.py`
**Как работает:**
1. Вызывается через AJAX (JavaScript с фронтенда).
2. Проверяется, есть ли кука. Принимаются ID вина и опциональное имя.
3. Валидируется переданный `wine_id` (принадлежит ли вино этой паре).
4. **Важный момент:** Проверяется модель `Vote`. Если голос от этой cookie для этой пары уже есть в БД — происходит **смена голоса**.
   - Очки у старого вина отнимаются: `pairing.votes_wine1 -= 1`.
   - Очки новому вину добавляются.
5. Запись сохраняется в базу: `pairing.save()`.

---

## 5. Завершение пары и раунда
**Функция:** `end_pairing(request, competition_id, pairing_id)`
**Как работает:**
1. Организатор нажимает "Завершить пару".
2. Окно голосования закрывается. Система проверяет: `если votes_wine1 > votes_wine2`, то победитель `wine1`.
3. **Обработка ничьей:** Если у вин равное количество голосов, то скрипт бросает жребий: `pairing.winner = random.choice([pairing.wine1, pairing.wine2])`. *(Если вам захочется сделать ручное определение победителя при ничьей — код нужно менять именно здесь).*
4. Статус пары меняется на `completed`.
5. **Вызов следующего шага:** Если эта пара была *последней незавершенной* в этом раунде, система автоматически вызывает закрытие раунда — `end_round()`.

**Функция:** `end_round(request, competition_id, round_id)`
**Как работает:**
1. Проходится по всем парам раунда: пропускает те, которые уже `completed` (результат их подсчета надежно зафиксирован).
2. Вычисляет всех оставшихся победителей `remaining_wines = [p.winner ...]`.
3. Если остался всего 1 участник — соревнование объявляется оконченным: `competition.status = "finished"`.
4. В противном случае — создает новый `Round` и снова вызывает `create_pairings()` для генерации списка новых пар из числа оставшихся.
