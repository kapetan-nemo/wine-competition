# Идея сайта

Сайт нужен для проведения дегустации с голосованием за наиболее понравившееся вино по олимпийской системе. На каждом этапе сравнивается N пар вин, из пары одно вино проходит дальше, и так до тех пор, пока не останется одно, которое и объявляется победителем. 
На сайте может одновременно проводиться разное количество соревнований. 

# Пользователи сайта

Пользователи сайта делятся на две категории. 

Администратор создает соревнование, описывает его характеристики – название, дата, сколько вин участвует, как они называются, какие-то особенности типа картинок, страны происхождения и т.д. 

Он же запускает соревнование, запускает начало и конец выбора среди одной пары и так далее. 

Пользователи заходят на страницу соревнования, отдают свой голос во время текущего голосования и могут смотреть статистику. 

На главной странице должна быть возможность выбрать роль – пользователь или организатор. 

## Реализация

Всё это надо реализовать в отдельном виртуальном окружении, чтобы не затрагивать содержимое компьютера, на котором происходит разработка. 
Кроме того, нужно создавать отдельные документации для описания деплоя, чтобы можно было запустить сайт на отдельном хостинге.

## Локальный запуск (через venv)

### Вариант 1. Автоматический скрипт

В проекте есть скрипт `scripts/run_local.sh`, который сам:

- создаёт (если нужно) виртуальное окружение в `.venv_run`,
- устанавливает зависимости из `requirements.txt`,
- применяет миграции,
- пробует создать суперпользователя `admin`,
- создаёт демо-соревнование,
- запускает сервер разработки.

Запуск из корня репозитория:

```bash
./scripts/run_local.sh
```

После этого сайт будет доступен по адресу `http://127.0.0.1:8000/`.

### Вариант 2. Ручной запуск

1. Создать и активировать виртуальное окружение (например, `.venv_run`):

```bash
python3 -m venv .venv_run
source .venv_run/bin/activate
```

2. Установить зависимости:

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

3. Применить миграции:

```bash
python manage.py migrate
```

4. (Опционально) создать суперпользователя:

```bash
python manage.py createsuperuser
```

5. Создать демо-данные:

```bash
python manage.py create_demo
```

6. Запустить сервер разработки:

```bash
python manage.py runserver 0.0.0.0:8000
```

### Как остановить сервер

#### Автоматический скрипт (`run_local.sh`)

Скрипт запускает сервер **в переднем плане** (foreground). Чтобы остановить его:

- Нажмите **Ctrl + C** в терминале, где работает сервер.

Если терминал недоступен, найдите и завершите процесс вручную:

```bash
# Найти PID процесса Django
lsof -ti :8000

# Завершить его
kill $(lsof -ti :8000)
```

#### Ручной запуск

Аналогично — нажмите **Ctrl + C** в терминале с запущенным `manage.py runserver`.
